# Coai

**A construction AI operating system for field capture, site execution, and project governance.**

Coai is being built to help construction teams capture site reality, review it properly, route it clearly, and carry it forward with stronger project truth, accountability, and continuity.

It starts with the operational loop between **Coai Field** and **Site Console**, then expands upward with discipline into **Main OS**, **Platform Console**, department-facing routing, and governed document layers such as **Vault**.

## Why Coai

Construction work still breaks when reality is split across calls, chats, paper notes, spreadsheets, photos, memory, and repeated explanation.

Coai is being built to reduce that structural gap with a clearer operating logic across:

- frontline capture
- single-project review and coordination
- cross-project routing and governance
- structural control, continuity, and document truth

## System direction

Coai is being shaped as a layered operating system:

- **Coai Field** — frontline capture with minimal friction
- **Site Console** — single-project review, coordination, and project truth
- **Main OS** — cross-project routing, departments, and governance
- **Platform Console** — identity, permissions, continuity, diagnostics, and structural control
- **Vault** — controlled documents, governed records, and document truth
- **Executive Console** — a staged later-phase oversight layer

## Current focus

The current beta focus is the **Coai Field ↔ Site Console** operating loop.

This is where workflow loss is immediate, visible, and operationally costly.  
The build starts there first, then expands upward without blurring authority boundaries or overstating maturity.

## Principles

Coai is being built around a few hard principles:

- **Truth over noise**
- **Capture before bureaucracy**
- **Review before authority**
- **Structure over scatter**
- **Governance without theatre**
- **Intelligence, kept in its place**
