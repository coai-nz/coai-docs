# Public Build Boundary

Coai is being built in public where visibility strengthens understanding, trust, and serious conversation.

## What may be public

Public-facing layers may include:

- product direction
- architecture notes
- system principles
- roadmap notes
- selected public-facing documentation
- selected build updates
- selected community-facing layers

## What may remain private

Some parts of Coai may remain private while the system matures, including:

- core commercial logic
- sensitive governance flows
- production-critical internals
- private operational implementation details
- security-sensitive system paths
- restricted business logic

## Why this boundary exists

Coai is not being built as openness theatre.

Public build is intended to make the direction, structure, and seriousness of the system visible.
Private boundaries remain where operational, commercial, and governance discipline require them.

## Current position

`coai-docs` is the public documentation and architecture-facing layer for Coai.

It exists to help aligned builders, pilot partners, strategic collaborators, and investors understand what is being built and why.
